# Contributing to EZ Pass WV

Thank you for your interest in contributing to the EZ Pass WV project!  
We value community involvement and appreciate your effort to improve this application.

---

## Contribution Guidelines

- Write clean, readable, and well-documented code
- Follow existing project structure and naming conventions
- Ensure your changes do not break existing features
- Test your code before submitting

---

## How to Contribute

1. Fork the repository
2. Clone your fork to your local machine
3. Create a new branch for your feature or fix
4. Make your changes and commit them
5. Push your branch to GitHub
6. Open a Pull Request to the main branch

---

## Reporting Issues

If you find a bug or unexpected behavior:
- Open an issue with a clear title
- Describe the steps to reproduce the issue
- Include screenshots or logs if possible

---

## Suggesting Features

We welcome feature suggestions!
- Open a new issue
- Clearly describe the feature
- Explain why it would be useful

---

## Code of Conduct

- Be respectful and inclusive
- Avoid offensive language
- Collaborate constructively
- Support fellow contributors

---

## Review Process

- Maintainers will review pull requests
- Feedback may be requested
- Approved changes will be merged into the main branch

Thank you for contributing to EZ Pass WV!
